## 一、漏洞分类定义

| 漏洞类型 | 定义 | 威胁 |
|---------|------|------|
| **水平越权** (Horizontal) | 同权限级别的用户 A 可以访问/操作用户 B 的资源 | 查看/修改他人隐私数据 |
| **垂直越权** (Vertical) | 低权限用户执行高权限用户才能做的操作 | 普通用户获得管理员权限 |
| **业务逻辑漏洞** | 程序逻辑的业务规则存在缺陷 | 绕过业务限制，非法获利 |

## 二、漏洞版本测试（修复前）

### 测试环境

系统中存在三个用户：`admin`（admin 角色）、`alice`（user 角色）、`bob`（注册用户）。

SQLite 表 `users` 没有 `role` 字段，所有用户存储在同一表中，`/profile` 和 `/recharge` 路由强制从 `request.args` / `request.form` 获取 `user_id`，不做权限校验：

```python
# 漏洞代码
@app.route("/profile")
def profile():
    user_id = request.args.get("user_id", "")  # ← 从URL获取，谁都能传
    row = conn.execute("SELECT ... FROM users WHERE id = ?", (user_id,)).fetchone()
    return render_template("profile.html", profile_user=dict(row))

@app.route("/recharge", methods=["POST"])
@csrf.exempt
def recharge():
    user_id = request.form.get("user_id", "")  # ← 从表单获取，谁都能传
    amount = request.form.get("amount", "0")   # ← 不校验正负
    conn.execute("UPDATE users SET balance = balance + ? WHERE id = ?", (amount, user_id))
```

## 三、修复过程

### 3.1 修复策略

| 漏洞 | 修复方案 | 原理 |
|------|---------|------|
| 水平越权 | 从 session 获取当前用户，不再信任 URL/表单中的 user_id | `_get_current_sqlite_user()` 基于 session["username"] 查询 |
| 未认证访问 | 添加 `if not session.get("username"): return redirect("/login")` | 每个路由入口验证登录状态 |
| 负数充值 | `if amount_val <= 0: error` | 充值金额必须为正数 |
| 超大金额 | `if amount_val > 100000: error` | 单次充值上限 10 万 |
| 非数字金额 | `float(amount)` 失败时返回明确错误 | 不再静默处理 |

### 3.2 修复前后代码对比

#### /profile — 修复前

```python
@app.route("/profile")
def profile():
    user_id = request.args.get("user_id", "")  # ← 谁都能传
    row = conn.execute("SELECT ... WHERE id = ?", (user_id,)).fetchone()
    return render_template("profile.html", profile_user=dict(row))
```

#### /profile — 修复后

```python
def _get_current_sqlite_user():
    """根据 session 中的 username 查询 SQLite 用户"""
    username = session.get("username")
    if not username:
        return None
    row = conn.execute("SELECT ... WHERE username = ?", (username,)).fetchone()
    return dict(row) if row else None

@app.route("/profile")
def profile():
    # 修复1: 必须登录
    if not session.get("username"):
        return redirect(url_for("login"))
    # 修复2: 从 session 获取当前用户, URL 参数被忽略
    current_user = _get_current_sqlite_user()
    return render_template("profile.html", profile_user=current_user)
```

#### /recharge — 修复前

```python
@app.route("/recharge", methods=["POST"])
@csrf.exempt
def recharge():
    user_id = request.form.get("user_id", "")  # ← 谁都能传
    amount = request.form.get("amount", "0")   # ← 不校验
    try:    amount_val = int(amount)
    except: amount_val = 0                     # ← 静默处理
    conn.execute("UPDATE users SET balance = balance + ? WHERE id = ?", (amount_val, user_id))
```

#### /recharge — 修复后

```python
@app.route("/recharge", methods=["POST"])
@csrf.exempt
def recharge():
    # 修复1: 必须登录
    if not session.get("username"):
        return redirect(url_for("login"))
    # 修复2: 只能给自己充值
    current_user = _get_current_sqlite_user()
    user_id = str(current_user["id"])

    amount = request.form.get("amount", "0")
    # 修复3: 必须为正数
    try:    amount_val = float(amount)
    except: return error("请输入有效的充值金额")
    if amount_val <= 0:    return error("充值金额必须大于0")
    if amount_val > 100000: return error("单次充值不能超过 ¥100,000")

    conn.execute("UPDATE users SET balance = balance + ? WHERE id = ?", (amount_val, user_id))
```

## 七、安全设计原则

1. **永远不要信任客户端传来的用户标识** — user_id 必须从服务端 token/session 推导
2. **每个操作前验证身份+权限** — 先验证"我是谁"，再检查"我有权做什么"
3. **业务输入必须有约束** — 金额必须有上下限、必须为正数
4. **错误必须明确** — 不要静默处理异常输入
